# @提及功能使用说明

## 功能概述

已成功实现类似哔哩哔哩的@提及功能，支持在帖子和评论中@关注的用户，并发送通知。

## 已实现的功能

### 1. 后端功能 ✅

#### 核心实体和服务
- **Mention实体**：存储@提及记录
  - 包含帖子ID、评论ID、被@用户ID、发起者ID
  - 内容预览（前200字符）
  - 已读状态
  - 创建时间

- **MentionService**：@业务逻辑
  - 提取内容中的`@[uid:xxx]`标记
  - 创建@提及记录
  - 查询@我的列表
  - 标记已读/删除通知
  - 获取未读数量

- **UserSearchService**：用户搜索
  - 只搜索当前用户关注的用户
  - 支持关键词过滤
  - 最多返回10个结果

#### API接口
- `GET /api/mentions/search?keyword=xxx` - 搜索关注的用户
- `GET /api/mentions/me?page=1&size=20` - 获取@我的列表
- `POST /api/mentions/{id}/read` - 标记已读
- `DELETE /api/mentions/{id}` - 删除通知
- `GET /api/mentions/unread-count` - 获取未读数量
- `GET /api/messages/mentions` - 兼容消息中心路径

#### 自动集成
- **PostService**：发布帖子时自动处理@
- **CommentService**：发表评论/回复时自动处理@

## 使用方式

### 在帖子中@某人

1. **方法一：使用@按钮**
   - 在帖子编辑器工具栏点击"@提及"按钮
   - 在弹出的列表中搜索或选择用户
   - 点击用户卡片，自动插入`@[uid:xxx]`标记

2. **方法二：手动输入（评论区）**
   - 在评论输入框输入`@`符号
   - 自动弹出关注的用户列表
   - 输入关键词搜索
   - 点击用户完成@操作

### 查看@我的消息

1. 进入消息中心
2. 点击左侧导航的"@我的"
3. 查看所有@通知
4. 点击通知跳转到对应帖子
5. 可以删除不需要的通知

### @标记格式

- **存储格式**：`@[uid:123]`
- **显示效果**：高亮的`@用户名`
- **同步更新**：用户名更改后自动显示最新名称（因为存储的是UID）

## 技术细节

### UID格式
- 使用`@[uid:xxx]`格式存储
- 正则表达式：`@\[uid:(\d+)\]`
- 解析时提取数字作为用户ID

### 防抖优化
- 搜索输入采用300ms防抖
- 避免频繁请求后端API

### 权限控制
- 只能@关注的用户
- 不能@自己
- 需要登录才能使用@功能

### 数据库表
```sql
CREATE TABLE mentions (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY,
    post_id BIGINT NOT NULL,
    comment_id BIGINT,
    mentioned_user_id BIGINT NOT NULL,
    mentioner_id BIGINT NOT NULL,
    content_preview VARCHAR(200),
    is_read BOOLEAN NOT NULL,
    create_time TIMESTAMP(6) NOT NULL,
    PRIMARY KEY (id)
);
```

## 注意事项

1. **首次使用需要先关注用户**
   - @功能只能@已关注的用户
   - 如果列表为空，请先关注一些用户

2. **富文本编辑器限制**
   - PostEditor使用wangeditor
   - @功能通过按钮触发，而非输入@符号
   - 这是为了简化富文本编辑器的集成复杂度

3. **评论区的@功能**
   - 输入@符号即可触发
   - 支持键盘快捷键Ctrl+Enter发送

4. **通知去重**
   - 同一条内容中多次@同一用户，只会创建一条通知
